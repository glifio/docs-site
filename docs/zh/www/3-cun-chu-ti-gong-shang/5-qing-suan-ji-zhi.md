# 清算机制

## 管理借款与清算价值比 (DTL) ，防止被清算

## 清算阈值：借款与清算价值比 (DTL) > 85%

[借款与清算价值比 (DTL)](/3-cun-chu-ti-gong-shang/3-jie-kuan-shang-xian-jie-kuan-yu-qing-suan-jia-zhi-bi-dtl) 用于衡量任何 SP 在 GLIF 系统中的资金安全，因为它代表了 SP 的抵押品情况。借款与清算价值比 (DTL)  < 100% 代表 SP 是过度抵押的，因为清算预计将收回 SP 欠 GLIF 池子的所有利息和本金。换句话说，清算将收回 SP 欠流动性提供者 (LP) 多于 100% 的资产。

债务对清算价值比 (DTL) 高于 85% 的 SP 处于清算的高度风险区。必须立即采取纠正措施，并与 GLIF 团队主动沟通，以避免被清算和最大限度地减少进一步损失。

在清算事件中，预计所有利息和本金将代表 LP 偿还给池子。如果在清算后还有剩余的 FIL，GLIF 金库将在欠池子的总利息 + 本金之外收取 10% 的清算费用。在 LP 和 GLIF 金库都得到偿还后，任何剩余 FIL 将退还给 SP。

> [!NOTE]
> 为了存储提供商的安全，存储提供商清算的过程目前由核心 GLIF 团队手动执行。一旦新的 Filecoin 改进提案通过并设置到核心 Filecoin 协议中，由算法控制的清算很快会推出。

## GLIF 管理状态 : DTL > 75%

除了暂停从 GLIF 借款和提取外，DTL 比率高于 75% 的 SP 还有机会被列入管理状态，这允许 GLIF 使用 SP Miner Actor 里任何可用余额来偿还 SP 的债务。如果 GLIF 管理员偿还 SP 的债务，扇区不会被终止或罚款。管理员的目标是让 SP 恢复至健康状态（DTL < 75%）。

## 健康 SP 的标准

1. 借款与清算价值比 (DTL)  < 75%
2. 所有活跃扇区中错误扇区少于 12%

如果 SP 的借款与清算价值比 (DTL) 比率超过 75%，它不会立即面临被清算的危险。但是，SP 有机会接受管理状态。如果 SP 的借款与清算价值比 (DTL) 超过 85%，它就有被清算的危险。强烈建议任何 SP 的借款与清算价值比 (DTL) 超过 75% 时，请立即联系 GLIF 团队。
